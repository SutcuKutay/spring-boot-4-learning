SECTION 1 - SPRING BOOT 4 QUICK START
---------------------
Spring Boot Actuator:
- Exposes endpoints to monitor and manage your application
- You easily get DevOps functionality out-of-the-box
- Simply add the dependency to your POM file
- REST endpoints are automatically added to your application
Endpoints are prefixed with: /actuator
/health -> Health information about your application
By default, only /health is exposed.
To expose /info type this in application.properties
	management.endpoints.web.exposure.include=health,info (* expose all endpoints)
	!!! management.endpoints.web.exposure.exclude=health,info (No one can access to these endpoints) !!!
	management.info.env.enabled=true
Update application.properties with your app info
	info.app.name=My Super Cool App
	info.app.description=A crazy and fun app, yoohoo!
	info.app.version=1.0.0
Properties starting with "info" will be used by /info
There are 20+ Spring Boot Actuator endpoints
- /auditevents -> Audit events for your application
- /beans -> List of all beans registered in the Spring application context
- /mappings -> List of all @RequestMapping paths
Full list: www.luv2code.com/actuator-endpoints

Spring Security:
You can override default user name and generated password:
application.properties ->
	spring.security.user.name=scott
	spring.security.user.password=tiger

Run from command-line:
Two choices,
	- java -jar
	- mvnw spring-boot:run
!!! If correct version of Maven is NOT found on your computer, use mvnw. Else just mvn is enough. /mvnw comes with a Sprin Boot project. !!!

SECTION 2 - SPRING CORE
---------------------
Inversion of Control (IoC)
The approach of outsourcing the construction and management of objects.

Dependency Injection
The client delegates to another object the responsibility of providing its dependencies.

Spring Container
Primary functions:
	- Create and manage objects (Inversion of Control)
	- Inject object's dependencies (Dependency Injection)

Injection Types
Two recommended types of injection:
	- Constructor Injection
	- Setter Injection

@SpringBootApplication (main java class) is composed of the following annotations:
	- @EnableAutoConfiguration -> Enables Spring Boot's auto-configuration support.
	- @ComponentScan -> Enables component scanning of current package. Also recursively scans sub-packages.
	- @Configuration -> Able to register extra beans with @Bean or import other configuration classes.

@Qualifier
@Primary -> Out of the implementations this is the primary implementations that should Spring Boot use.
Qualifier has higher priority.

Lazy Initialization
By default, when your application starts, all beans are initialized. Spring will create an instance of each and make
them available.
Instead of creating all beans up front, we can specify lazy initialization. A bean will only be initialized in the following cases:
	- It is needed for dependency injection
	- Or it is explicitly requested
Add the @Lazy annotation to a given class
OR
Add this line to application.properties file for global configuration:
	spring.main.lazy-initialization=true

SECTION 4 - REST CRUD APIs
---------------------
REST: REpresentational State Transfer
Lightweight approach for communicating between applications

Java JSON Data Binding
Data binding is the process of converting JSON data to a Java POJO
Spring uses the Jackson Project behind the scenes
Jackson handles data binding between JSON and Java POJO

REST Exception Handling
	1- Create a custom error response class
	2- Create a custom exception class
	3- Update REST service to throw exception if student not found
	4- Add an exception handler using @ExceptionHandler
For global;
	1- Create new @ControllerAdvice
	2- Refactor REST service ... remove exception handling code
	3- Add exception handling code to @ControllerAdvice

REST API Design - Best Practices
	1- Review API requirements
	2- Identify main resource / entity
	3- Use HTTP methods to assign action on resource
1-REST clients should be able to
-Get a list of employees
-Get a single employee by id
-Add a new employee
-Update an employee
-Delete an employee
2-To identify main resource / entity, look for the most prominent "noun"
For our project, it is "employee"
Convention is to use plural form of resource / entity: employees
/api/employees
3-
POST -> Create a new entity -> /api/employees
GET -> Read a list of entities or single entity -> /api/employees or /api/employees/{employeeId}
PUT -> Update an existing entity -> /api/employees
DELETE -> Delete an existing entity -> /api/employees/{employeeId}

Spring Data REST
Leverages your existing JpaRepository
Spring will give you a REST CRUD implementation for FREE ... like MAGIC!!!
Simple pluralized form
	First character of Entity type is lowercase
	Then just adds an "s" to the entity
		JpaRepository<Employee, Integer> -> /employees
Only need 3 items:
	- Your entity: Employee
	- JpaRepository: EmployeeRepository extends JpaRepository<Employee, Integer>
	- Maven POM dependency for: spring-boot-starter-data-rest
For UPDATE, Spring Data REST only uses ID on the URL, /employees/4 dont put "id" on JSON request body
@RepositoryRestResource(path="members")
public interface EmployeeRepository extends JpaRepository... -> custom path name. This way Spring does not automatically generate a "employees" path.
By default, Spring Data REST will return the first 20 elements
	Page size = 20
You can navigate to the different pages of data using query param
	http://localhost:8080/employees?page=0
	http://localhost:8080/employees?page=1 Pages are zero-based
Following properties available: application.properties
	-spring.data.rest.base-path -> Base path used to expose repository resources
	-spring.data.rest.default-page-size -> Default size of pages
	-spring.data.rest.max-page-size -> Maximum size of pages
Sort by last name
	http://localhost:8080/employees?sort=lastName,firstName,desc (default asc)
