SECTION 1 - SPRING BOOT 4 QUICK START
---------------------
Spring Boot Actuator:
- Exposes endpoints to monitor and manage your application
- You easily get DevOps functionality out-of-the-box
- Simply add the dependency to your POM file
- REST endpoints are automatically added to your application
Endpoints are prefixed with: /actuator
/health -> Health information about your application
By default, only /health is exposed.
To expose /info type this in application.properties
	management.endpoints.web.exposure.include=health,info (* expose all endpoints)
	!!! management.endpoints.web.exposure.exclude=health,info (No one can access to these endpoints) !!!
	management.info.env.enabled=true
Update application.properties with your app info
	info.app.name=My Super Cool App
	info.app.description=A crazy and fun app, yoohoo!
	info.app.version=1.0.0
Properties starting with "info" will be used by /info
There are 20+ Spring Boot Actuator endpoints
- /auditevents -> Audit events for your application
- /beans -> List of all beans registered in the Spring application context
- /mappings -> List of all @RequestMapping paths
Full list: www.luv2code.com/actuator-endpoints

Spring Security:
You can override default user name and generated password:
application.properties ->
	spring.security.user.name=scott
	spring.security.user.password=tiger

Run from command-line:
Two choices,
	- java -jar
	- mvnw spring-boot:run
!!! If correct version of Maven is NOT found on your computer, use mvnw. Else just mvn is enough. /mvnw comes with a Sprin Boot project. !!!

Custom Application Properties:
Default located at: src/main/resources/application.properties
Spring Boot can access properties using @Value annotation

Spring Boot Properties:
List of Common Properties
	www.luv2code.com/spring-boot-props
Set logging levels based on package names
	Inside application.properties file
		logging.level.org.springframework=DEBUG
		logging.level.org.hibernate=TRACE
		logging.level.com.luv2code=INFO
Logging levels are: TRACE, DEBUG, INFO, WARN, ERROR, FATAL, OFF
		logging.file.name=my-crazy-stuff.log
		logging.file.path=c:/myapps/demo
Spring Boot Logging -> www.luv2code.com/spring-boot-logging
Web properties
		server.port=7070
		server.servlet.context-path=/my-silly-app (default context path is /)
		server.servlet.session.timeout=15m (default timeout 30 minutes)
Actuator properties
		management.endpoints.web.exposure.include=*
		management.endpoints.web.exposure.exclude=beans,mapping
		management.endpoints.web.base-path=/actuator
Security properties
		spring.security.user.name=admin
		spring.security.user.password=topsecret
Data properties
		spring.datasource.url=jdbc:mysql://localhost:3306/ecommerce
		spring.datasource.username=scott
		spring.datasource.password=tiger

SECTION 2 - SPRING CORE
---------------------
Inversion of Control (IoC)
The approach of outsourcing the construction and management of objects.

Dependency Injection
The client delegates to another object the responsibility of providing its dependencies.

Spring Container
Primary functions:
	- Create and manage objects (Inversion of Control)
	- Inject object's dependencies (Dependency Injection)

Injection Types
Two recommended types of injection:
	- Constructor Injection
	- Setter Injection

@SpringBootApplication (main java class) is composed of the following annotations:
	- @EnableAutoConfiguration -> Enables Spring Boot's auto-configuration support.
	- @ComponentScan -> Enables component scanning of current package. Also recursively scans sub-packages.
	- @Configuration -> Able to register extra beans with @Bean or import other configuration classes.

@Qualifier
@Primary -> Out of the implementations this is the primary implementations that should Spring Boot use.
Qualifier has higher priority.

Lazy Initialization
By default, when your application starts, all beans are initialized. Spring will create an instance of each and make
them available.
Instead of creating all beans up front, we can specify lazy initialization. A bean will only be initialized in the following cases:
	- It is needed for dependency injection
	- Or it is explicitly requested
Add the @Lazy annotation to a given class
OR
Add this line to application.properties file for global configuration:
	spring.main.lazy-initialization=true

What is Hibernate?

Your Java App <-> Hibernate <-> Database

What is JPA?
Jakarta Persistence API(JPA)
